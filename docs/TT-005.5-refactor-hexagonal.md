# TT-005.5: Refactor de Arquitectura Hexagonal

## Descripción
Corrección de violaciones estructurales de la arquitectura hexagonal identificadas en una revisión de código. Se implementó Inversión de Dependencias real mediante inyección por constructor, se agregaron tipos de dominio, y se eliminaron los acoplamientos directos entre capas.

## Violaciones Detectadas y Corregidas

| # | Problema | Gravedad | Solución |
|---|----------|:--------:|----------|
| a | `PropiedadService` importaba `propiedadRepository` concreto (singleton) | 🔴 Alta | Constructor recibe `IPropiedadRepository` |
| b | `IPropiedadRepository` importaba `Prisma` del generado | 🔴 Alta | Se eliminó el import (no se usaba) |
| c | `PropiedadRepository` no usaba `implements IPropiedadRepository` | 🟡 Media | Se agregó `implements` a ambas clases |
| d | Todo el flujo usaba `any`, sin entidades de dominio | 🟡 Media | Se crearon `domain/entities/Propiedad.ts` y `Asesor.ts` |
| e | Singletons exportados impedían testear con mocks | 🔴 Alta | Servicios reciben repositorio por constructor |

## Nueva Estructura de Dependencias

```
routes/asesor.routes.ts          ← Único punto de wiring
  ├── new AsesorRepository()     ← infraestructura
  ├── new AsesorService(repo)    ← aplicación (solo conoce IAsesorRepository)
  └── new AsesorController(svc)  ← infraestructura

routes/propiedad.routes.ts        ← Único punto de wiring
  ├── new PropiedadRepository()   ← infraestructura
  ├── new PropiedadService(repo)  ← aplicación (solo conoce IPropiedadRepository)
  └── new PropiedadController(svc) ← infraestructura
```

Ninguna capa interna (domain, application) conoce las implementaciones concretas de infraestructura.

## Beneficios

- **Testeable:** Se puede pasar un mock del repositorio al servicio en pruebas unitarias
- **Desacoplado:** Cambiar Prisma por otro ORM solo requiere tocar el repositorio (una clase)
- **Verificable:** El compilador TypeScript valida que `PropiedadRepository implements IPropiedadRepository`
- **Dominio aislado:** `domain/` no importa nada de `infrastructure/` ni de librerías externas
